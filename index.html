BREAST CANCER PREDICTION PROJECT¶

In this project, I will make use of measurements of cell nucleus from a breast mass to classify the mass as benign or malignant breast cancer. The data set was obtained from the kaggle website. Also can be found on UCI Machine Learning Repository: https://archive.ics.uci.edu/ml/datasets/Breast+Cancer+Wisconsin+%28Diagnostic%29.

Features Description:

  • id : ID number
  • Diagnosis: (M = malignant, B = benign)
  • columns 3-32: measurements of cell nucleus

Ten real-valued features computed for each cell nucleus:

a) radius (mean of distances from center to points on the perimeter) b) texture (standard deviation of gray-scale values) c) perimeter d) area e) smoothness (local variation in radius lengths) f) compactness (perimeter^2 / area - 1.0) g) concavity (severity of concave portions of the contour) h) concave points (number of concave portions of the contour) i) symmetry j) fractal dimension ("coastline approximation" - 1)

The mean, standard error and "worst" or largest (mean of the three largest values) of these features were computed for each image, resulting in 30 features. For instance, field 3 is Mean Radius, field 13 is Radius SE, field 23 is Worst Radius.

The dataset is comprised of 569 observations, and 32 columns.

As seen in the correlation matrix above, the following features has near-zero correlation with diagnosis and will not be used in the classification model:

- fractal_dimension_mean, texture_se, smoothness_se, symmetry_se, and fractal_dimension_se.

Also, the following groups of features are highly correlated with each other.

- radius mean, perimeter mean, area mean, radius worst, perimeter worst, area worst
- texture mean, texture worst
- concavity mean, concave points mean, concave points worst
- radius se, perimeter se, area se

To resolve the multicollinearity, I chose one variable from each group that had the highest correlation with diagnosis. The correlation table shows the correlation each of the colinear variables has with diagnosis. I chose the following features from each group to be included in the classification model, and dropped the other features in the groups:

 - perimeter worst
 - texture worst
 - concave points worst
 - radius se

def plot_curve_LogR():
    # instantiate
    LogR = LogisticRegression(penalty='l2', C=1, fit_intercept=True, random_state=123, solver='liblinear')
    
    # fit
    X = data_X
    y = data_y
    LogR.fit(X, y)
    

    train_sizes, train_err, val_err = learning_curve(LogR, X, y, n_jobs=1, cv=10, train_sizes=np.linspace(.1, 1.0, 5), verbose=0,
                                                    scoring = "neg_mean_squared_error")

    train_err = abs(train_err)**0.5
    val_err = abs(val_err)**0.5
    
    
    train_error_mean = np.mean(train_err, axis=1)
    train_error_std = np.std(train_err, axis=1)
    val_error_mean = np.mean(val_err, axis=1)
    val_error_std = np.std(val_err, axis=1)
    
    plt.figure()
    plt.title("Logistic Regression Model")
   
    # plot the std deviation as a transparent range at each training set size
    plt.fill_between(train_sizes, train_error_mean - train_error_std, train_error_mean + train_error_std, alpha=0.1, color="r")
    plt.fill_between(train_sizes, val_error_mean - val_error_std, val_error_mean + val_error_std, alpha=0.1, color="g")
    
    # plot the average training and test score lines at each training set size
    plt.plot(train_sizes, train_error_mean, 'o-', color="r", label="Training RMSE")
    plt.plot(train_sizes, val_error_mean, 'o-', color="g", label="Cross-validation RMSE")
    plt.legend(loc='lower right')
    plt.xlabel("Training examples")
    plt.ylabel("Root Mean Squared Error")
    
    
    # box-like grid
    plt.grid()
    # sizes the window for readability and displays the plot
    # shows error from 0 to 1.1
   
    plt.show()
